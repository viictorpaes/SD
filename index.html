Plague

Desafie a sorte: um jogo de tensão crescente onde cartas definem seu destino e um botão errado aciona o alarme.

Ver no GitHub

Um jogo estilo 'Croc Croc'. Serão uma sequência de botões e os jogadores puxarão cartas cada um para definir quantos botões apertar. Na rodada 1 será um botão infectado, na 2 dois botões e assim sucessivamente. Quando o botão correto for pressionado um led verde acenderá e quando o infectado for pressionado um led vermelho seguido de um beep para indicar que o jogador perdeu.

Motivação

A motivação para o desenvolvimento deste projeto reside na oportunidade de aplicar, de forma prática e lúdica, conceitos fundamentais de eletrônica digital e lógica de programação. O projeto desafia a capacidade de integrar hardware (sensores e atuadores) com software, criando um sistema capaz de gerenciar estados complexos e aleatoriedade.

Objetivo

-Criar um jogo interativo

-Proporcionar uma experiência divertida

-Ser acessível para qualquer público

Contextualização

Este projeto propõe o desenvolvimento de um dispositivo de entretenimento eletrônico interativo, inspirado na mecânica clássica de jogos de sorte e tensão (popularmente conhecido como estilo "Croc Croc" ou Crocodile Dentist).

Diferente da versão puramente mecânica do brinquedo tradicional, este protótipo utiliza lógica de programação e hardware para criar uma experiência dinâmica. O sistema substitui dentes físicos por uma matriz de botões e introduz novas camadas de complexidade ao jogo:

Aleatoriedade Controlada: O uso de cartas para determinar a quantidade de ações por jogador.
Dificuldade Progressiva: Um sistema de rodadas onde a probabilidade de erro aumenta exponencialmente (1 botão "infectado" na primeira rodada, 2 na segunda, etc.).
Feedback Sensorial: Respostas audiovisuais imediatas (LEDs verdes/vermelhos e avisos sonoros) para validar as jogadas.

O objetivo é criar uma experiência de "Party Game" moderna, onde a eletrônica gerencia o acaso, garantindo imparcialidade e aumentando a tensão entre os participantes a cada rodada.

Passo a Passo

1. Materiais Necessários

Para seguir este guia, você precisará de:

  • 1x Placa Arduino (Uno, Nano ou similar).
  • 1x Protoboard.
  • 3x LEDs RGB (do tipo cátodo comum). Nota: se não tiver RGB, pode usar 3 LEDs vermelhos e 3 verdes separados.
  • 4x Botões (3 para o jogo, 1 para Reset).
  • 4x Resistores de 10kΩ (pull-down para os botões).
  • 3x a 6x Resistores de 220Ω ou 330Ω (para os LEDs).
  • 1x Buzzer (ativo ou passivo).
  • Jumpers (fios) nas cores: Vermelho, Verde, Azul, Preto, Laranja e Cinza.

2. Montagem do Circuito (Hardware)

Vamos montar seguindo a lógica do código e as cores dos fios sugeridas abaixo.

Passo A: Alimentação (Fios Laranja e Preto)

  • Laranja (5V): Conecte o pino 5V do Arduino na trilha positiva (+) da protoboard.
  • Preto (GND): Conecte o pino GND do Arduino na trilha negativa (-) da protoboard.

Passo B: LEDs RGB (Conjuntos 1, 2 e 3)

Os LEDs RGB têm 4 pernas; a mais longa é o cátodo comum (GND).

  • Conecte a perna comum de todos os LEDs ao GND (trilha negativa) da protoboard.
  • Use resistores de 220Ω entre cada pino do Arduino e a perna do LED para proteção.
  • Conexões sugeridas:
    • Conjunto 1: pino 9 → perna vermelha; pino 8 → perna verde.
    • Conjunto 2: pino 11 → perna vermelha; pino 10 → perna verde.
    • Conjunto 3: pino 13 → perna vermelha; pino 12 → perna verde.

Passo C: Botões de Jogo (Fios Azuis)

Configuração com pull-down (resistor para GND):

  • Um lado do botão → 5V (trilha positiva).
  • O outro lado do botão: uma perna com resistor de 10kΩ para GND e um jumper para o pino do Arduino.
  • Conexões dos botões:
    • Botão 1 → pino D2
    • Botão 2 → pino D3
    • Botão 3 → pino D4

Passo D: Botão de Reset

Segue a mesma lógica dos botões de jogo. Conecte a saída do botão ao pino D5 do Arduino.

Passo E: Buzzer (Fio Cinza)

  • Fio Cinza: pino D6 → perna positiva (+) do buzzer.
  • Fio Preto: perna negativa (–) do buzzer → GND.

3. Software (Upload do Código)

  1. Abra a Arduino IDE no computador.
  2. Conecte o Arduino via cabo USB.
  3. Em Ferramentas > Placa, selecione seu modelo (ex.: Arduino Uno).
  4. Em Ferramentas > Porta, selecione a porta COM que apareceu.
  5. Cole o código do sketch (o trecho disponível na seção Código) e clique em Carregar.

5. Como Testar

  1. Ligue o Arduino; o sketch executará o setup() e sorteará um número (graças ao randomSeed).
  2. Aperte um dos botões (1, 2 ou 3):
    • Se acender um LED verde → jogada válida (salvo).
    • Se acender um LED vermelho + beep → botão "infectado" (jogador perde).
  3. Para jogar novamente, aperte o botão Reset (pino D5) para apagar LEDs e sortear novo "infectado".

Código

Abaixo está o código completo utilizado no projeto:

//Declarção das entradas:
int vermPin1 = 9;     // Pino do LED vermelho do primeiro conjunto
int verdPin1 = 8;     // Pino do LED verde do primeiro conjunto
int btnPin1  = 2;     // Pino do botão do primeiro conjunto

int vermPin2 = 11;    // Pino do LED vermelho do segundo conjunto
int verdPin2 = 10;    // Pino do LED verde do segundo conjunto
int btnPin2  = 3;     // Pino do botão do segundo conjunto

int vermPin3 = 13;    // Pino do LED vermelho do terceiro conjunto
int verdPin3 = 12;    // Pino do LED verde do terceiro conjunto
int btnPin3  = 4;     // Pino do botão do terceiro conjunto

int btnReset = 5;     // Pino do botão de reset

int buzzer = 6;       // Pino do buzzer

int sorteio;          // Variável que armazena o número aleatório (1 a 3)

void setup() {  //essa função é rodada pelo arduino uma unica vez antes de iniciar o código
  Serial.begin(9600);           // Inicializa comunicação serial

  pinMode(vermPin1, OUTPUT);    // Define o LED vermelho 1 como saída
  pinMode(verdPin1, OUTPUT);    // Define o LED verde 1 como saída
  pinMode(vermPin2, OUTPUT);    // Define o LED vermelho 2 como saída
  pinMode(verdPin2, OUTPUT);    // Define o LED verde 2 como saída
  pinMode(vermPin3, OUTPUT);    // Define o LED vermelho 3 como saída
  pinMode(verdPin3, OUTPUT);    // Define o LED verde 3 como saída

  pinMode(btnPin1, INPUT);      // Define o botão 1 como entrada
  pinMode(btnPin2, INPUT);      // Define o botão 2 como entrada
  pinMode(btnPin3, INPUT);      // Define o botão 3 como entrada
  
  pinMode(buzzer, OUTPUT);      // Define o buzzer como saída

  pinMode(btnReset, INPUT);     // Define o botão de reset como entrada

  randomSeed(analogRead(A0));   // Gera semente aleatória 
  sorteio = random(1, 4);       // Sorteia um número de 1 a 3
}

void desligaTodos() { //função que eu criei para usar depois
  digitalWrite(vermPin1, LOW);  // Desliga LED vermelho 1
  digitalWrite(verdPin1, LOW);  // Desliga LED verde 1
  digitalWrite(vermPin2, LOW);  // Desliga LED vermelho 2
  digitalWrite(verdPin2, LOW);  // Desliga LED verde 2
  digitalWrite(vermPin3, LOW);  // Desliga LED vermelho 3
  digitalWrite(verdPin3, LOW);  // Desliga LED verde 3

  randomSeed(analogRead(A0));   // Atualiza a semente aleatória
  sorteio = random(1, 4);       // Sorteia novamente um número entre 1 e 3
}

void loop() { //essa função o arduino fica rodando de novo e de novo

  int b1 = digitalRead(btnPin1);   // Lê estado do botão 1
  int b2 = digitalRead(btnPin2);   // Lê estado do botão 2
  int b3 = digitalRead(btnPin3);   // Lê estado do botão 3
  int bReset = digitalRead(btnReset); // Lê estado do botão de reset

  if (bReset == HIGH) {           // Se o botão de reset for pressionado
    desligaTodos();               // Desliga todos LEDs e faz novo sorteio
  }

  // Caso o número sorteado seja 1
  if (sorteio == 1) {

    if (b1 == HIGH){              // Se o botão 1 foi pressionado
        digitalWrite(vermPin1, HIGH); // Liga LED vermelho 1
        tone(buzzer, 1500);           // Aciona o buzzer
        delay(300);                   // Aguarda 300 ms
        noTone(buzzer);               // Desliga o buzzer
    }

    if (b2 == HIGH) digitalWrite(verdPin2, HIGH); // Pressão no botão 2 → LED verde 2 liga
    if (b3 == HIGH) digitalWrite(verdPin3, HIGH); // Pressão no botão 3 → LED verde 3 liga
  }

  // Caso o número sorteado seja 2
  else if (sorteio == 2) {

    if (b1 == HIGH) digitalWrite(verdPin1, HIGH); // Pressão no botão 1 → LED verde 1 liga

    if (b2 == HIGH){                // Pressão no botão 2
        digitalWrite(vermPin2, HIGH); // Liga LED vermelho 2
        tone(buzzer, 1500);           // Toca o buzzer
        delay(300);
        noTone(buzzer);
    }

    if (b3 == HIGH) digitalWrite(verdPin3, HIGH); // Pressão no botão 3 → LED verde 3 liga
  }

  // Caso o número sorteado seja 3
  else if (sorteio == 3) {

    if (b1 == HIGH) digitalWrite(verdPin1, HIGH); // Pressão no botão 1 → LED verde 1 liga
    if (b2 == HIGH) digitalWrite(verdPin2, HIGH); // Pressão no botão 2 → LED verde 2 liga

    if (b3 == HIGH){                // Pressão no botão 3
        digitalWrite(vermPin3, HIGH); // Liga LED vermelho 3
        tone(buzzer, 1500);
        delay(300);
        noTone(buzzer);
    }
  }
}

Problemas enfrentados e lições aprendidas

Desafios

  1. Garantia de Aleatoriedade (True Randomness)

    O Problema: Inicialmente, o microcontrolador tendia a repetir a mesma sequência de números "sorteados" toda vez que era ligado, pois a função random() é pseudo-aleatória. Isso tornava o jogo previsível.

    A Solução: Foi implementada a função randomSeed(analogRead(A0)) no setup() e na função de reset. Ao ler uma porta analógica desconectada (flutuante), o sistema captura ruído estático do ambiente para gerar uma semente única, garantindo que o botão infectado seja imprevisível a cada nova rodada.

  2. Organização do Circuito e "Espaguete de Fios"

    O Problema: Com 3 conjuntos de LEDs, 4 botões e um buzzer, a protoboard ficou saturada, dificultando a identificação de erros de conexão e causando instabilidade nos sinais (mau contato).

    A Solução: Adotou-se um padrão rigoroso de codificação por cores para a fiação (Vermelho/Verde para saídas de LED, Azul para entradas de botões, Laranja/Preto para alimentação). Além disso, o código foi reestruturado utilizando variáveis descritivas para os pinos, facilitando a manutenção do hardware.

  3. Lógica de Reinício (Reset do Jogo)

    O Problema: A necessidade de reiniciar o jogo sem desligar a energia do Arduino.

    A Solução: Criação de uma função dedicada (desligaTodos()) e a implementação de um botão físico de Reset. Essa função zera os estados dos LEDs e força um novo sorteio imediatamente, permitindo jogabilidade contínua.

Lições aprendidas

  • Integração Hardware-Software: A importância de alinhar a lógica de programação com a montagem física. Um erro no código pode parecer falha no componente e vice-versa.
  • Modularização do Código: O uso de funções específicas (como a desligaTodos()) torna o código mais limpo, legível e fácil de debugar do que escrever tudo dentro do loop().
  • Feedback ao Usuário: Indicadores visuais (LEDs) e sonoros (buzzer) são essenciais para confirmar imediatamente se a ação foi registrada e qual foi o resultado.
  • Planejamento de Hardware: A complexidade da montagem aumentou a percepção sobre a necessidade de planejar o espaço na protoboard e a distribuição de energia antes de iniciar a conexão dos fios.
